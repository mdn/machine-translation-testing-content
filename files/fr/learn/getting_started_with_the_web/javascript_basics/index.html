JavaScript est un langage de programmation qui ajoute de l'interactivité à votre site Web. Cela se produit dans les jeux, dans le comportement des réponses lorsque vous appuyez sur les boutons ou lors de la saisie de données sur les formulaires; avec un style dynamique; avec des animations, etc. Cet article vous aide à démarrer avec JavaScript et vous aide à mieux comprendre ce qui est possible.

Qu'est-ce que JavaScript?

{{Glossary("JavaScript")}} («JS» en abrégé) est un véritable{{Glossary("Dynamic programming language", "dynamic programming language")}} qui peut ajouter de l'interactivité à un site Web. Il a été inventé par Brendan Eich (co-fondateur du projet Mozilla, de la Mozilla Foundation et de la Mozilla Corporation).

JavaScript est polyvalent et convivial pour les débutants. Avec plus d'expérience, vous serez en mesure de créer des jeux, des graphiques 2D et 3D animés, des applications complètes basées sur une base de données et bien plus encore!

JavaScript lui-même est relativement compact, mais très flexible. Les développeurs ont écrit une variété d'outils en plus du langage JavaScript de base, débloquant une grande quantité de fonctionnalités avec un minimum d'effort. Ceux-ci inclus:

  • Interfaces de programmation d'application de navigateur ({{Glossary("API","APIs")}} ) intégré aux navigateurs Web, fournissant des fonctionnalités telles que la création dynamique de HTML et la définition de styles CSS; collecter et manipuler un flux vidéo à partir de la webcam d'un utilisateur, ou générer des graphiques 3D et des échantillons audio.
  • API tierces qui permettent aux développeurs d'incorporer des fonctionnalités dans des sites d'autres fournisseurs de contenu, tels que Twitter ou Facebook.
  • Des frameworks et bibliothèques tiers que vous pouvez appliquer au HTML pour accélérer le travail de création de sites et d'applications.

Cet article n'entre pas dans le cadre de cet article - en tant qu'introduction légère à JavaScript - pour présenter les détails de la différence entre le langage JavaScript principal et les outils répertoriés ci-dessus. Vous pouvez en savoir plus dans la zone d'apprentissage JavaScript de MDN, ainsi que dans d'autres parties de MDN.

La section ci-dessous présente certains aspects du langage de base et offre également la possibilité de jouer avec quelques fonctionnalités de l'API du navigateur. S'amuser!

Un bonjour tout le monde! exemple

JavaScript est l'une des technologies Web modernes les plus populaires! Au fur et à mesure que vos compétences JavaScript se développeront, vos sites Web entreront dans une nouvelle dimension de puissance et de créativité.

Cependant, se familiariser avec JavaScript est plus difficile que de se familiariser avec HTML et CSS. Vous devrez peut-être commencer petit et progresser progressivement. Pour commencer, examinons comment ajouter JavaScript à votre page pour créer un monde Hello! exemple. ( Hello world! Est la norme pour les exemples de programmation d'introduction .)

Important : si vous n'avez pas suivi le reste de notre cours, téléchargez cet exemple de code et utilisez-le comme point de départ.

  1. Accédez à votre site de test et créez un nouveau dossier nommé scripts . Dans le dossier scripts, créez un nouveau fichier appelé main.js et enregistrez-le.
  2. Dans votre fichier index.html , entrez ce code sur une nouvelle ligne, juste avant la </body> fermeture </body> : 
    <script src="scripts/main.js"></script>
  3. Cela fait le même travail que le{{htmlelement("link")}} élément pour CSS. Il applique le JavaScript à la page, de sorte qu'il peut avoir un effet sur le HTML (avec le CSS, et tout autre élément sur la page).
  4. Ajoutez ce code au fichier main.js : 
    const myHeading = document.querySelector('h1');
myHeading.textContent = 'Hello world!';
  5. Assurez-vous que les fichiers HTML et JavaScript sont enregistrés. Ensuite, chargez index.html dans votre navigateur. Vous devriez voir quelque chose comme ceci:

Remarque : la raison pour laquelle les instructions (ci-dessus) placent le{{htmlelement("script")}} L'élément près du bas du fichier HTML est que le navigateur lit le code dans l'ordre dans lequel il apparaît dans le fichier .

Si le JavaScript se charge en premier et qu'il est censé affecter le HTML qui n'a pas encore été chargé, il pourrait y avoir des problèmes. Placer JavaScript au bas d'une page HTML est un moyen de gérer cette dépendance. Pour en savoir plus sur les approches alternatives, consultez Stratégies de chargement de script .

Qu'est-il arrivé?

Le texte de l'en-tête est devenu Bonjour tout le monde! en utilisant JavaScript. Vous avez fait cela en utilisant une fonction appelée{{domxref("Document.querySelector", "querySelector()")}} pour saisir une référence à votre en-tête, puis la stocker dans une variable appelée myHeading . Ceci est similaire à ce que nous avons fait en utilisant les sélecteurs CSS. Lorsque vous voulez faire quelque chose sur un élément, vous devez d'abord le sélectionner.

Ensuite, le code définit la valeur de la variable myHeading{{domxref("Node.textContent", "textContent")}} propriété (qui représente le contenu de l'en-tête) à Hello world! .

Remarque : les deux fonctionnalités que vous avez utilisées dans cet exercice font partie de l' API DOM (Document Object Model) , qui a la capacité de manipuler des documents.

Cours accéléré sur les bases de la langue

Pour vous donner une meilleure compréhension du fonctionnement de JavaScript, expliquons certaines des fonctionnalités de base du langage. Il est à noter que ces fonctionnalités sont communes à tous les langages de programmation. Si vous maîtrisez ces principes de base, vous avez également une longueur d'avance sur le codage dans d'autres langues!

Important : dans cet article, essayez de saisir les exemples de lignes de code dans votre console JavaScript pour voir ce qui se passe. Pour plus d'informations sur les consoles JavaScript, voir Découvrir les outils de développement de navigateur .

Variables

{{Glossary("Variable", "Variables")}} sont des conteneurs qui stockent des valeurs. Vous commencez par déclarer une variable avec le var (moins recommandé, approfondissez pour l'explication) ou le mot let clé let , suivi du nom que vous donnez à la variable: 

let myVariable;

Remarque : un point-virgule à la fin d'une ligne indique où se termine une instruction. Il n'est requis que lorsque vous devez séparer les instructions sur une seule ligne. Cependant, certaines personnes pensent que c'est une bonne pratique d'avoir des points-virgules à la fin de chaque instruction. Il existe d'autres règles pour savoir quand vous devriez et ne devriez pas utiliser des points-virgules. Pour plus de détails, consultez Votre guide des points-virgules en JavaScript .

Remarque : vous pouvez nommer une variable à peu près n'importe quoi, mais il existe certaines restrictions. (Consultez cette section sur les règles de dénomination .) Si vous n'êtes pas sûr, vous pouvez vérifier le nom de votre variable pour voir s'il est valide.

Remarque : JavaScript est sensible à la casse. Cela signifie que myVariable n'est pas la même chose que myvariable . Si vous rencontrez des problèmes dans votre code, vérifiez le cas!

Remarque : pour plus de détails sur la différence entre var et let , voir Différence entre var et let .

Après avoir déclaré une variable, vous pouvez lui donner une valeur: 

myVariable = 'Bob';

Vous pouvez également effectuer ces deux opérations sur la même ligne: 

let myVariable = 'Bob';

Vous récupérez la valeur en appelant le nom de la variable: 

myVariable;

Après avoir attribué une valeur à une variable, vous pouvez la modifier ultérieurement dans le code: 

let myVariable = 'Bob';
myVariable = 'Steve';

Notez que les variables peuvent contenir des valeurs qui ont différents types de données :

Variable Explication Exemple
{{Glossary("String")}} Il s'agit d'une séquence de texte appelée chaîne. Pour indiquer que la valeur est une chaîne, placez-la entre guillemets simples. let myVariable = 'Bob';
{{Glossary("Number")}} C'est un nombre. Les nombres n'ont pas de guillemets autour d'eux. let myVariable = 10;
{{Glossary("Boolean")}} Il s'agit d'une valeur True / False. Les mots true et false sont des mots clés spéciaux qui n'ont pas besoin de guillemets. let myVariable = true;
{{Glossary("Array")}} Il s'agit d'une structure qui vous permet de stocker plusieurs valeurs dans une seule référence. let myVariable = [1,'Bob','Steve',10];
Reportez-vous à chaque membre du tableau comme ceci:
myVariable[0] , myVariable[1] , etc.
{{Glossary("Object")}} Cela peut être n'importe quoi. Tout en JavaScript est un objet et peut être stocké dans une variable. Gardez cela à l'esprit pendant que vous apprenez. let myVariable = document.querySelector('h1');
Tous les exemples ci-dessus aussi.

Alors pourquoi avons-nous besoin de variables? Les variables sont nécessaires pour faire quelque chose d'intéressant en programmation. Si les valeurs ne peuvent pas changer, vous ne pouvez rien faire de dynamique, comme personnaliser un message d'accueil ou modifier une image affichée dans une galerie d'images.

commentaires

Les commentaires sont des extraits de texte qui peuvent être ajoutés avec du code. Le navigateur ignore le texte marqué comme commentaires. Vous pouvez écrire des commentaires en JavaScript comme vous le pouvez en CSS: 

/*
Everything in between is a comment.
*/

Si votre commentaire ne contient pas de saut de ligne, c'est une option pour le mettre derrière deux barres obliques comme ceci: 

// This is a comment

Les opérateurs

Un{{Glossary("operator")}} est un symbole mathématique qui produit un résultat basé sur deux valeurs (ou variables). Dans le tableau suivant, vous pouvez voir certains des opérateurs les plus simples, ainsi que quelques exemples à essayer dans la console JavaScript.

Opérateur Explication Symbole (s) Exemple
Une addition Ajoutez deux nombres ensemble ou combinez deux chaînes. + 6 + 9;
'Hello ' + 'world!';
Soustraction, multiplication, division Ceux-ci font ce que vous attendez d'eux en mathématiques de base. - , * , / 9 - 3;
8 * 2; // multiply in JS is an asterisk
9 / 3;
Affectation Comme vous l'avez déjà vu: cela attribue une valeur à une variable. = let myVariable = 'Bob';
Égalité Cela effectue un test pour voir si deux valeurs sont égales. Il renvoie un résultat true / false (booléen). === let myVariable = 3;
myVariable === 4;
Non, n'est pas égal Cela renvoie la valeur logiquement opposée de ce qu'il précède. Il transforme un true en false , etc. Lorsqu'il est utilisé avec l'opérateur Égalité, l'opérateur de négation teste si deux valeurs ne sont pas égales. ! , !==

Pour "Not", l'expression de base est true , mais la comparaison renvoie false car nous la nions:

let myVariable = 3;
!(myVariable === 3);

"N'est pas égal" donne fondamentalement le même résultat avec une syntaxe différente. Ici, nous testons "est myVariable PAS égale à 3". Cela renvoie false car myVariable EST égale à 3:

let myVariable = 3;
myVariable !== 3;

Il y a beaucoup plus d'opérateurs à explorer, mais cela suffit pour l'instant. Voir Expressions et opérateurs pour une liste complète.

Remarque : le mélange de types de données peut conduire à des résultats étranges lors de l'exécution de calculs. Veillez à vous référer correctement à vos variables et à obtenir les résultats attendus. Par exemple, entrez '35' + '25' dans votre console. Pourquoi n'obtenez-vous pas le résultat attendu? Comme les guillemets transforment les nombres en chaînes, vous avez fini par concaténer des chaînes plutôt que d'ajouter des nombres. Si vous entrez 35 + 25 vous obtiendrez le total des deux nombres.

Conditionnels

Les conditions sont des structures de code utilisées pour tester si une expression renvoie vrai ou non. Une forme très courante de conditions est l'instruction if ... else . Par exemple:

 let iceCream = 'chocolate';
if(iceCream === 'chocolate') {
  alert('Yay, I love chocolate ice cream!');
} else {
  alert('Awwww, but chocolate is my favorite...');
}

L'expression à l'intérieur du if( ... ) est le test. Cela utilise l'opérateur d'identité (comme décrit ci-dessus) pour comparer la variable iceCream avec la chaîne de chocolate pour voir si les deux sont égaux. Si cette comparaison renvoie true , le premier bloc de code s'exécute. Si la comparaison n'est pas vraie, le deuxième bloc de code - après l'instruction else - s'exécute à la place.

Les fonctions

{{Glossary("Function", "Functions")}} sont une manière de conditionner les fonctionnalités que vous souhaitez réutiliser. Il est possible de définir un corps de code comme une fonction qui s'exécute lorsque vous appelez le nom de la fonction dans votre code. C'est une bonne alternative à l'écriture répétée du même code. Vous avez déjà vu quelques utilisations des fonctions précédemment. Par exemple:

  1.  let myVariable = document.querySelector('h1');
  2. alert('hello!');

Ces fonctions, document.querySelector et alert , sont intégrées au navigateur.

Si vous voyez quelque chose qui ressemble à un nom de variable, mais qui est suivi de parenthèses - () - c'est probablement une fonction. Les fonctions prennent souvent{{Glossary("Argument", "arguments")}} : bits de données dont ils ont besoin pour faire leur travail. Les arguments sont placés entre parenthèses, séparés par des virgules s'il y a plus d'un argument.

Par exemple, la fonction alert() fait apparaître une boîte de dialogue dans la fenêtre du navigateur, mais nous devons lui donner une chaîne comme argument pour indiquer à la fonction quel message afficher.

Vous pouvez également définir vos propres fonctions. Dans l'exemple suivant, nous créons une fonction simple qui prend deux nombres comme arguments et les multiplie: 

function multiply(num1,num2) {
  let result = num1 * num2;
  return result;
}

Essayez de l'exécuter dans la console; puis testez avec plusieurs arguments. Par exemple: 

multiply(4, 7);
multiply(20, 20);
multiply(0.5, 3);

Remarque : l'instruction return indique au navigateur de renvoyer la variable de result hors de la fonction afin qu'elle soit disponible à l'utilisation. Ceci est nécessaire car les variables définies à l'intérieur des fonctions ne sont disponibles qu'à l'intérieur de ces fonctions. C'est ce qu'on appelle la variable{{Glossary("Scope", "scoping")}} . (En savoir plus sur la portée des variables .)

Événements

La véritable interactivité sur un site Web nécessite des gestionnaires d'événements. Ce sont des structures de code qui écoutent l'activité dans le navigateur et exécutent du code en réponse. L'exemple le plus évident est la gestion de l' événement de clic , qui est déclenché par le navigateur lorsque vous cliquez sur quelque chose avec votre souris. Pour démontrer cela, entrez ce qui suit dans votre console, puis cliquez sur la page Web actuelle: 

document.querySelector('html').onclick = function() {
    alert('Ouch! Stop poking me!');
}

Il existe de nombreuses façons d'attacher un gestionnaire d'événements à un élément. Ici, nous sélectionnons le{{htmlelement("html")}} élément, définissant sa propriété onclick handler égale à une fonction anonyme (c'est-à-dire sans nom), qui contient le code que nous voulons que l'événement click s'exécute.

Notez que 

document.querySelector('html').onclick = function() {};

est équivalent à

 let myHTML = document.querySelector('html');
myHTML.onclick = function() {};

C'est juste plus court.

Suralimentation de notre site Web d'exemple

Une fois cet examen des bases de JavaScript terminé (ci-dessus), ajoutons de nouvelles fonctionnalités à notre site d'exemple.

Ajout d'un changeur d'images

Dans cette section, vous apprendrez à utiliser les fonctionnalités de JavaScript et de l'API DOM pour alterner l'affichage de l'une des deux images. Ce changement se produira lorsqu'un utilisateur clique sur l'image affichée.

  1. Choisissez une image que vous souhaitez afficher sur votre site d'exemple. Idéalement, l'image sera de la même taille que l'image que vous avez ajoutée précédemment, ou aussi proche que possible.
  2. Enregistrez cette image dans votre dossier images .
  3. Renommez l'image firefox2.png .
  4. Ajoutez le JavaScript ci-dessous à votre fichier main.js (Supprimez également votre JavaScript Hello world! De l'exercice précédent.)
     let myImage = document.querySelector('img');

myImage.onclick = function() {
    let mySrc = myImage.getAttribute('src');
    if(mySrc === 'images/firefox-icon.png') {
      myImage.setAttribute('src','images/firefox2.png');
    } else {
      myImage.setAttribute('src','images/firefox-icon.png');
    }
}
  5. Enregistrez tous les fichiers et chargez index.html dans le navigateur. Maintenant, lorsque vous cliquez sur l'image, elle devrait changer pour l'autre.

C'est ce qui s'est passé. Vous avez enregistré une référence à votre{{htmlelement("img")}} élément dans la variable myImage . Ensuite, vous avez rendu la propriété de onclick événement onclick cette variable égale à une fonction sans nom (une fonction «anonyme»). Ainsi, chaque fois que vous cliquez sur cet élément:

  1. Le code récupère la valeur de l'attribut src de l'image.
  2. Le code utilise un conditionnel pour vérifier si la valeur src est égale au chemin de l'image d'origine:
    1. Si tel est le cas, le code change la valeur src en chemin de la deuxième image, forçant l'autre image à être chargée dans le{{htmlelement("img")}} élément.
    2. Si ce n'est pas le cas (ce qui signifie qu'elle doit déjà avoir changé), la valeur src revient au chemin de l'image d'origine, à l'état d'origine.

Ajouter un message de bienvenue personnalisé

Ensuite, changeons le titre de la page en un message de bienvenue personnalisé lorsque l'utilisateur visite le site pour la première fois. Ce message de bienvenue persistera. Si l'utilisateur quitte le site et y revient plus tard, nous enregistrerons le message à l'aide de l' API Web Storage . Nous inclurons également une option pour changer l'utilisateur, et donc, le message de bienvenue.

  1. Dans index.html , ajoutez la ligne suivante juste avant le{{htmlelement("script")}} élément: 
    <button>Change user</button>
  2. Dans main.js , placez le code suivant au bas du fichier, exactement tel qu'il est écrit. Cela prend des références au nouveau bouton et à l'en-tête, en stockant chaque variable interne:
     let myButton = document.querySelector('button');
let myHeading = document.querySelector('h1');
  3. Ajoutez la fonction ci-dessous pour définir le message d'accueil personnalisé. Cela ne fera rien pour le moment, mais cela changera bientôt. 
    function setUserName() {
  let myName = prompt('Please enter your name.');
  localStorage.setItem('name', myName);
  myHeading.textContent = 'Mozilla is cool, ' + myName;
}
    La fonction setUserName() contient une fonction prompt() , qui affiche une boîte de dialogue, similaire à alert() . Cette fonction prompt() fait plus que alert() , demandant à l'utilisateur d'entrer des données et de les stocker dans une variable après que l'utilisateur clique sur OK. Dans ce cas, nous demandons à l'utilisateur de saisir un nom. Ensuite, le code fait appel à une API localStorage , ce qui nous permet de stocker des données dans le navigateur et de les récupérer plus tard. Nous utilisons la fonction setItem() de localStorage pour créer et stocker un élément de données appelé 'name' , en définissant sa valeur sur la variable myName qui contient l'entrée de l'utilisateur pour le nom. Enfin, nous définissons le textContent de l'en-tête sur une chaîne, plus le nom nouvellement stocké de l'utilisateur.
  4. Ajoutez le bloc if ... else (ci-dessous). Nous pourrions appeler ce code d'initialisation, car il structure l'application lors de son premier chargement. 
    if(!localStorage.getItem('name')) {
  setUserName();
} else {
  let storedName = localStorage.getItem('name');
  myHeading.textContent = 'Mozilla is cool, ' + storedName;
}
    Cette première ligne de ce bloc utilise l'opérateur de négation (NON logique, représenté par le ! ) Pour vérifier si les données de name existent. Sinon, la fonction setUserName() s'exécute pour le créer. S'il existe (c'est-à-dire que l'utilisateur a défini un nom d'utilisateur lors d'une visite précédente), nous getItem() nom stocké à l'aide de getItem() et définissons le textContent de l'en-tête sur une chaîne, plus le nom de l'utilisateur, comme nous l'avons fait dans setUserName() .
  5. Placez ce onclick événements onclick (ci-dessous) sur le bouton. Lorsque vous cliquez dessus, setUserName() s'exécute. Cela permet à l'utilisateur d'entrer un nom différent en appuyant sur le bouton. 
    myButton.onclick = function() {
  setUserName();
}

Un nom d'utilisateur nul?

Lorsque vous exécutez l'exemple et obtenez la boîte de dialogue qui vous invite à entrer votre nom d'utilisateur, essayez d'appuyer sur le bouton Annuler . Vous devriez vous retrouver avec un titre qui indique que Mozilla est cool, nul . Cela se produit car, lorsque vous annulez l'invite, la valeur est définie sur null . Null est une valeur spéciale en JavaScript qui fait référence à l'absence de valeur.

Essayez également de cliquer sur OK sans entrer de nom. Vous devriez vous retrouver avec un titre qui dit que Mozilla est cool, pour des raisons assez évidentes.

Pour éviter ces problèmes, vous pouvez vérifier que l'utilisateur n'a pas entré de nom vide. Mettez à jour votre fonction setUserName() à ceci: 

function setUserName() {
  let myName = prompt('Please enter your name.');
  if(!myName) {
    setUserName();
  } else {
    localStorage.setItem('name', myName);
    myHeading.innerHTML = 'Mozilla is cool, ' + myName;
  }
}

En langage humain, cela signifie: si myName n'a pas de valeur, exécutez à nouveau setUserName() depuis le début. S'il a une valeur (si l'instruction ci-dessus n'est pas vraie), stockez la valeur dans localStorage et définissez-la comme texte de l'en-tête.
