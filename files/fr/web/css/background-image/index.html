{{CSSRef}}

La propriété CSS background-image définit une ou plusieurs images d'arrière-plan sur un élément.

{{EmbedInteractiveExample("pages/css/background-image.html")}}
La source de cet exemple interactif est stockée dans un référentiel GitHub. Si vous souhaitez contribuer au projet d'exemples interactifs, veuillez cloner https://github.com/mdn/interactive-examples et nous envoyer une pull request.

Les images d'arrière-plan sont dessinées sur des couches de contexte empilées les unes sur les autres. Le premier calque spécifié est dessiné comme s'il était le plus proche de l'utilisateur.

Les bordures de l'élément sont ensuite dessinées au-dessus d'eux, et le{{cssxref("background-color")}} est dessiné sous eux. La façon dont les images sont dessinées par rapport à la boîte et à ses bordures est définie par le{{cssxref("background-clip")}} et{{cssxref("background-origin")}} Propriétés CSS.

Si une image spécifiée ne peut pas être dessinée (par exemple, lorsque le fichier indiqué par l'URI spécifié ne peut pas être chargé), les navigateurs la traitent comme ils le feraient avec une valeur none .

Remarque: même si les images sont opaques et que la couleur ne s'affiche pas dans des circonstances normales, les développeurs Web doivent toujours spécifier un{{cssxref("background-color")}} . Si les images ne peuvent pas être chargées - par exemple, lorsque le réseau est en panne - la couleur d'arrière-plan sera utilisée comme solution de secours.

Syntaxe

Chaque image d'arrière-plan est spécifiée soit comme mot-clé none soit comme{{cssxref("<image>")}} valeur.

Pour spécifier plusieurs images d'arrière-plan, indiquez plusieurs valeurs, séparées par une virgule: 

background-image:
  linear-gradient(to bottom, rgba(255,255,0,0.5), rgba(0,0,255,0.5)),
  url('https://mdn.mozillademos.org/files/7693/catfront.png');

Valeurs

none
Est un mot clé dénotant l'absence d'images.
<image>
Est un{{cssxref("<image>")}} indiquant l'image à afficher. Il peut y en avoir plusieurs, séparés par des virgules, car plusieurs arrière - plans sont pris en charge.

Problèmes d'accessibilité

Les navigateurs ne fournissent aucune information particulière sur les images d'arrière-plan aux technologies d'assistance. Ceci est important avant tout pour les lecteurs d'écran, car un lecteur d'écran n'annoncera pas sa présence et ne transmettra donc rien à ses utilisateurs. Si l'image contient des informations essentielles pour comprendre l'objectif général de la page, il est préférable de la décrire sémantiquement dans le document.

Exemples

Superposition d'images d'arrière-plan

Notez que l'image de l'étoile est partiellement transparente et est superposée à l'image du chat.

HTML 

<div>
  <p class="catsandstars">
    This paragraph is full of cats<br />and stars.
  </p>
  <p>This paragraph is not.</p>
  <p class="catsandstars">
    Here are more cats for you.<br />Look at them!
  </p>
  <p>And no more.</p>
</div>

CSS 

p {
  font-size: 1.5em;
  color: #FE7F88;
  background-image: none;
  background-color: transparent;
}

div {
  background-image:
      url("https://mdn.mozillademos.org/files/6457/mdn_logo_only_color.png");
}

.catsandstars {
  background-image:
      url("https://mdn.mozillademos.org/files/11991/startransparent.gif"),
      url("https://mdn.mozillademos.org/files/7693/catfront.png");
  background-color: transparent;
}
