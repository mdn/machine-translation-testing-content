JavaScript是一種可向您的網站添加交互性的編程語言。這發生在遊戲中，按下按鈕時的響應行為或在表單上輸入數據時；具有動態樣式；動畫等。本文可幫助您開始使用JavaScript並加深對可能的理解。

什麼是JavaScript？

{{Glossary("JavaScript")}} （簡稱“ JS”）是成熟的{{Glossary("Dynamic programming language", "dynamic programming language")}}可以增加網站的互動性。它是由Brendan Eich（Mozilla項目，Mozilla基金會和Mozilla公司的聯合創始人）發明的。

JavaScript是通用的並且對初學者友好。有了更多的經驗，您將能夠創建遊戲，動畫2D和3D圖形，全面的數據庫驅動的應用程序等等！

JavaScript本身相對緊湊，但非常靈活。開發人員已在核心JavaScript語言之上編寫了各種工具，以最少的努力即可解鎖大量功能。這些包括：

  • 瀏覽器應用程序編程接口（{{Glossary("API","APIs")}} ）內置到網絡瀏覽器中，提供動態創建HTML和設置CSS樣式等功能；收集和處理來自用戶網絡攝像頭的視頻流，或生成3D圖形和音頻樣本。
  • 第三方API，允許開發人員在來自其他內容提供商（例如Twitter或Facebook）的站點中合併功能。
  • 您可以將其應用於HTML的第三方框架和庫，以加快構建站點和應用程序的工作。

作為JavaScript的簡要介紹，這超出了本文的討論範圍，無法提供有關JavaScript核心語言與上面列出的工具的不同之處的詳細信息。您可以在MDN的JavaScript學習區域以及MDN的其他部分中了解更多信息。

以下部分介紹了核心語言的某些方面，並提供了使用一些瀏覽器API功能的機會。玩得開心！

變數

{{Glossary("Variable", "Variables")}}是存儲值的容器。首先，使用var （不建議使用，請深入說明）或let關鍵字聲明變量，然後使用為變量指定的名稱： 

let myVariable;

注意：行尾的分號表示語句結束的位置。僅在需要在一行上分隔語句時才需要它。但是，有些人認為在每個語句的末尾使用分號是一種好習慣。對於何時應該使用和不應該使用分號還有其他規則。有關更多詳細信息，請參見《 JavaScript分號指南》 。

注意：您幾乎可以為變量命名，但是有一些限制。 （有關命名規則，請參閱本節。）如果不確定，可以檢查變量名是否有效。

注意：JavaScript區分大小寫。這意味著myVariable與myvariable 。如果您的代碼有問題，請檢查大小寫！

注意：有關var和let之間的區別的更多詳細信息，請參閱var和let之間的區別。

聲明變量後，可以給它一個值： 

myVariable = 'Bob';

另外，您可以在同一行上執行以下兩項操作： 

let myVariable = 'Bob';

您可以通過調用變量名稱來檢索值： 

myVariable;

為變量分配值後，您可以稍後在代碼中對其進行更改： 

let myVariable = 'Bob';
myVariable = 'Steve';

請注意，變量可能包含具有不同數據類型的值：

變量說明例
{{Glossary("String")}}這是稱為字符串的文本序列。要表示該值是一個字符串，請將其用單引號引起來。 let myVariable = 'Bob';
{{Glossary("Number")}}這是一個數字。數字周圍沒有引號。 let myVariable = 10;
{{Glossary("Boolean")}}這是一個True / False值。單詞true和false是不需要引號的特殊關鍵字。 let myVariable = true;
{{Glossary("Array")}}這種結構允許您在單個引用中存儲多個值。 let myVariable = [1,'Bob','Steve',10];
像這樣引用數組的每個成員：
myVariable[0] ， myVariable[1]等
{{Glossary("Object")}}這可以是任何東西。 JavaScript中的所有內容都是一個對象，可以存儲在變量中。學習時請記住這一點。 let myVariable = document.querySelector('h1');
以上所有示例也是如此。

那麼為什麼我們需要變量呢？變量是進行編程中任何有趣的事情所必需的。如果值不能更改，那麼您將無法做任何動態的事情，例如個性化問候消息或更改圖庫中顯示的圖像。

有條件的

條件是用於測試表達式是否返回true的代碼結構。條件語句的一種非常常見的形式是if ... else語句。例如：

 let iceCream = 'chocolate';
if(iceCream === 'chocolate') {
  alert('Yay, I love chocolate ice cream!');
} else {
  alert('Awwww, but chocolate is my favorite...');
}

if( ... )內部的表達式是測試。這使用身份運算符（如上所述）將變量iceCream與chocolate iceCream進行比較，以查看兩者是否相等。如果此比較返回true ，則運行第一段代碼。如果比較不正確，則運行else語句之後的第二個代碼塊。

職能

{{Glossary("Function", "Functions")}}是您希望重用的打包功能的一種方式。可以將代碼主體定義為在代碼中調用函數名稱時執行的函數。這是重複編寫相同代碼的好選擇。您以前已經看過函數的一些用法。例如：

  1.  let myVariable = document.querySelector('h1');
  2. alert('hello!');

這些功能document.querySelector和alert內置於瀏覽器中。

如果看到類似於變量名的名稱，但後面帶有括號()則可能是函數。功能通常需要{{Glossary("Argument", "arguments")}}：完成工作所需的數據位。引數放在括號內，如果有多個引數，則用逗號分隔。

例如， alert()函數使一個彈出框出現在瀏覽器窗口內，但是我們需要給它一個字符串作為參數，以告訴該函數顯示什麼消息。

您也可以定義自己的功能。在下一個示例中，我們創建一個簡單的函數，該函數將兩個數字作為參數並將它們相乘： 

function multiply(num1,num2) {
  let result = num1 * num2;
  return result;
}

嘗試在控制台中運行它；然後測試幾個參數。例如： 

multiply(4, 7);
multiply(20, 20);
multiply(0.5, 3);

注意： return語句告訴瀏覽器將result變量從函數中返回，以便可以使用。這是必需的，因為在函數內部定義的變量僅在這些函數內部可用。這稱為變量{{Glossary("Scope", "scoping")}}。 （了解有關變量作用域的更多信息。）

大事記

網站上的真正互動需要事件處理程序。這些是在瀏覽器中偵聽活動並作為響應運行代碼的代碼結構。最明顯的示例是處理click事件，當您用鼠標單擊某些內容時，瀏覽器會觸發該事件。為了說明這一點，請在控制台中輸入以下內容，然後單擊當前網頁： 

document.querySelector('html').onclick = function() {
    alert('Ouch! Stop poking me!');
}

將事件處理程序附加到元素的方法有很多。在這裡我們選擇{{htmlelement("html")}}元素，將其onclick處理程序屬性設置為等於匿名（即，無名）函數，該函數包含我們希望click事件運行的代碼。

注意

document.querySelector('html').onclick = function() {};

相當於

let myHTML = document.querySelector('html');
myHTML.onclick = function() {};

它只是更短。

添加圖像轉換器

在本部分中，您將學習如何使用JavaScript和DOM API功能來交替顯示兩個圖像之一。當用戶單擊顯示的圖像時，將發生此更改。

  1. 選擇您要在示例網站上展示的圖片。理想情況下，圖像的大小應與之前添加的圖像相同，或盡可能接近。
  2. 將此圖像保存在您的images文件夾中。
  3. 重命名圖像firefox2.png 。
  4. 將下面的JavaScript添加到您的main.js文件中。 （還可以從前面的練習中刪除Hello world！ JavaScript。）
     let myImage = document.querySelector('img');

myImage.onclick = function() {
    let mySrc = myImage.getAttribute('src');
    if(mySrc === 'images/firefox-icon.png') {
      myImage.setAttribute('src','images/firefox2.png');
    } else {
      myImage.setAttribute('src','images/firefox-icon.png');
    }
}
  5. 保存所有文件，然後在瀏覽器中加載index.html 。現在，當您單擊圖像時，它應該更改為另一幅圖像。

就是這樣您存儲了對您的參考{{htmlelement("img")}}myImage變量中的元素。接下來，使此變量的onclick事件處理程序屬性等於沒有名稱的函數（“匿名”函數）。因此，每次單擊此元素：

  1. 該代碼檢索圖像的src屬性的值。
  2. 該代碼使用條件檢查src值是否等於原始圖像的路徑：
    1. 如果是的話，代碼會將src值更改為第二張圖片的路徑，從而迫使另一張圖片加載到{{htmlelement("img")}}元件。
    2. 如果不是（表示必須已經更改），則src值會交換回原始圖像路徑，並恢復為原始狀態。

添加個性化歡迎消息

接下來，讓我們在用戶首次訪問該網站時將頁面標題更改為個性化的歡迎消息。此歡迎消息將繼續存在。如果用戶離開站點並稍後返回，我們將使用Web Storage API保存消息。我們還將包括一個用於更改用戶的選項，因此也提供了歡迎消息。

  1. 在index.html ， index.html下行添加到{{htmlelement("script")}}元件： 
    <button>Change user</button>
  2. 在main.js ，將以下代碼準確地放在文件底部。這將引用新按鈕和標題，並存儲每個內部變量：
     let myButton = document.querySelector('button');
let myHeading = document.querySelector('h1');
  3. 添加以下功能來設置個性化問候語。目前尚無任何作用，但很快就會改變。 
    function setUserName() {
  let myName = prompt('Please enter your name.');
  localStorage.setItem('name', myName);
  myHeading.textContent = 'Mozilla is cool, ' + myName;
}
    所述setUserName()函數包含一個prompt()函數，它顯示一個對話框，類似於alert()這個alert() prompt()函數的作用不只是alert() ，它還要求用戶輸入數據並將其存儲在用戶單擊OK之後的變量中。在這種情況下，我們要求用戶輸入名稱。接下來，代碼調用API localStorage ，這使我們可以將數據存儲在瀏覽器中並在以後檢索。我們使用localStorage的setItem()函數來創建和存儲名為'name'的數據項，並將其值設置為myName變量，該變量包含用戶輸入的名稱。最後，我們將標題的textContent設置為字符串，再加上用戶新存儲的名稱。
  4. 在下面添加if ... else塊。我們可以調用此初始化代碼，因為它在首次加載時會構建應用程序。 
    if(!localStorage.getItem('name')) {
  setUserName();
} else {
  let storedName = localStorage.getItem('name');
  myHeading.textContent = 'Mozilla is cool, ' + storedName;
}
    該塊的第一行使用否定運算符（邏輯NOT，由!表示）檢查name數據是否存在。如果不是，則運行setUserName()函數來創建它。如果存在（即用戶在上一次訪問期間設置了用戶名），我們將使用getItem()檢索存儲的名稱，並將標題的textContent設置為字符串，再加上用戶名，就像在setUserName()所做的setUserName() 。
  5. 將此onclick事件處理程序（如下）放在按鈕上。單擊後， setUserName()運行setUserName() 。這允許用戶通過按下按鈕輸入其他名稱。 
    myButton.onclick = function() {
  setUserName();
}

用戶名是否為null？

當您運行該示例並獲得提示您輸入用戶名的對話框時，請嘗試按“取消”按鈕。您應該以一個標題結尾，該標題顯示Mozilla很酷，null 。發生這種情況的原因是-當您取消提示時-該值設置為null 。 Null是JavaScript中的一個特殊值，它表示缺少值。

另外，嘗試單擊“確定”而不輸入名稱。由於相當明顯的原因，您應該以一個標題結尾，該標題的內容應為Mozilla很棒。

為避免這些問題，您可以檢查用戶是否輸入了空白名稱。將您的setUserName()函數更新為： 

function setUserName() {
  let myName = prompt('Please enter your name.');
  if(!myName) {
    setUserName();
  } else {
    localStorage.setItem('name', myName);
    myHeading.innerHTML = 'Mozilla is cool, ' + myName;
  }
}

用人類語言來說，這意味著：如果myName沒有值，請setUserName()開始再次運行setUserName() 。如果它確實有一個值（如果上面的語句不正確），則將該值存儲在localStorage並將其設置為標題的文本。
