JavaScript是一种可向您的网站添加交互性的编程语言。这发生在游戏中，按下按钮时的响应行为或在表单上输入数据时；具有动态样式；动画等。本文可帮助您开始使用JavaScript并加深对可能的理解。

什么是JavaScript？

{{Glossary("JavaScript")}} （简称“ JS”）是成熟的{{Glossary("Dynamic programming language", "dynamic programming language")}}可以增加网站的互动性。它是由Brendan Eich（Mozilla项目，Mozilla基金会和Mozilla公司的联合创始人）发明的。

JavaScript是通用的并且对初学者友好。有了更多的经验，您将能够创建游戏，动画2D和3D图形，全面的数据库驱动的应用程序等等！

JavaScript本身相对紧凑，但非常灵活。开发人员已在核心JavaScript语言之上编写了各种工具，以最少的努力即可解锁大量功能。这些包括：

  • 浏览器应用程序编程接口（{{Glossary("API","APIs")}} ）内置到网络浏览器中，提供动态创建HTML和设置CSS样式等功能；收集和处理来自用户网络摄像头的视频流，或生成3D图形和音频样本。
  • 第三方API，允许开发人员在来自其他内容提供商（例如Twitter或Facebook）的站点中合并功能。
  • 您可以将其应用于HTML的第三方框架和库，以加快构建站点和应用程序的工作。

作为JavaScript的简要介绍，这超出了本文的讨论范围，无法提供有关JavaScript核心语言与上面列出的工具的不同之处的详细信息。您可以在MDN的JavaScript学习区域以及MDN的其他部分中了解更多信息。

以下部分介绍了核心语言的某些方面，并提供了使用一些浏览器API功能的机会。玩得开心！

语言基础速成课程

为了让您更好地了解JavaScript的工作原理，让我们解释一下该语言的一些核心功能。值得注意的是，这些功能是所有编程语言所共有的。如果您掌握了这些基础知识，那么您也将拥有其他语言编码的先机！

重要提示：在本文中，请尝试将示例代码行输入到JavaScript控制台中，以查看会发生什么。有关JavaScript控制台的更多详细信息，请参阅发现浏览器开发人员工具。

变数

{{Glossary("Variable", "Variables")}}是存储值的容器。首先，使用var （不建议使用，请深入说明）或let关键字声明变量，然后使用为变量指定的名称： 

let myVariable;

注意：行尾的分号表示语句结束的位置。仅在需要在一行上分隔语句时才需要它。但是，有些人认为在每个语句的末尾使用分号是一种好习惯。对于何时应该使用和不应该使用分号还有其他规则。有关更多详细信息，请参见《 JavaScript分号指南》 。

注意：您几乎可以为变量命名，但是有一些限制。 （有关命名规则，请参阅本节。）如果不确定，可以检查变量名是否有效。

注意：JavaScript区分大小写。这意味着myVariable与myvariable 。如果您的代码有问题，请检查大小写！

注意：有关var和let之间的区别的更多详细信息，请参阅var和let之间的区别。

声明变量后，可以给它一个值： 

myVariable = 'Bob';

另外，您可以在同一行上执行以下两项操作： 

let myVariable = 'Bob';

您可以通过调用变量名称来检索值： 

myVariable;

为变量分配值后，您可以稍后在代码中对其进行更改： 

let myVariable = 'Bob';
myVariable = 'Steve';

请注意，变量可能包含具有不同数据类型的值：

变量说明例
{{Glossary("String")}}这是称为字符串的文本序列。要表示该值是一个字符串，请将其用单引号引起来。 let myVariable = 'Bob';
{{Glossary("Number")}}这是一个数字。数字周围没有引号。 let myVariable = 10;
{{Glossary("Boolean")}}这是一个True / False值。单词true和false是不需要引号的特殊关键字。 let myVariable = true;
{{Glossary("Array")}}这种结构允许您在单个引用中存储多个值。 let myVariable = [1,'Bob','Steve',10];
像这样引用数组的每个成员：
myVariable[0] ， myVariable[1]等
{{Glossary("Object")}}这可以是任何东西。 JavaScript中的所有内容都是一个对象，可以存储在变量中。学习时请记住这一点。 let myVariable = document.querySelector('h1');
以上所有示例也是如此。

那么为什么我们需要变量呢？变量是进行编程中任何有趣的事情所必需的。如果值不能更改，那么您将无法做任何动态的事情，例如个性化问候消息或更改图库中显示的图像。

一个{{Glossary("operator")}}是一个基于两个值（或变量）产生结果的数学符号。在下表中，您可以看到一些最简单的运算符，以及可以在JavaScript控制台中尝试的一些示例。

操作员说明符号例

'Hello ' + 'world!';

myVariable === 4;
不，不等于这将返回其先前值的逻辑相反值。它会变成true成false ，等等。当使用旁边的平等经营者，否定运算符测试两个值是否不相等。 ! ， !==

对于“ Not”，基本表达式为true ，但是比较返回false因为我们将其取反：

let myVariable = 3;
!(myVariable === 3);

“不等于”使用不同的语法给出基本相同的结果。在这里，我们正在测试“ myVariable不等于3”。由于myVariable IS等于3，因此返回false ：

注意：执行计算时，混合数据类型可能会导致一些奇怪的结果。请注意，您正确地引用了变量，并获得了预期的结果。例如，在控制台中输入'35' + '25' 。为什么没有得到预期的结果？由于引号将数字转换为字符串，因此您最终将字符串串联起来，而不是添加数字。如果输入35 + 25 ，则将得到两个数字的总和。

有条件的

条件是用于测试表达式是否返回true的代码结构。条件语句的一种非常常见的形式是if ... else语句。例如：

 let iceCream = 'chocolate';
if(iceCream === 'chocolate') {
  alert('Yay, I love chocolate ice cream!');
} else {
  alert('Awwww, but chocolate is my favorite...');
}

if( ... )内部的表达式是测试。这使用身份运算符（如上所述）将变量iceCream与chocolate iceCream进行比较，以查看两者是否相等。如果此比较返回true ，则运行第一段代码。如果比较不正确，则运行else语句之后的第二个代码块。

职能

{{Glossary("Function", "Functions")}}是您希望重用的打包功能的一种方式。可以将代码主体定义为在代码中调用函数名称时执行的函数。这是重复编写相同代码的好选择。您以前已经看过函数的一些用法。例如：

  1.  let myVariable = document.querySelector('h1');
  2. alert('hello!');

这些功能document.querySelector和alert内置于浏览器中。

如果看到类似于变量名的名称，但后面带有括号()则可能是函数。功能通常需要{{Glossary("Argument", "arguments")}}：完成工作所需的数据位。引数放在括号内，如果有多个引数，则用逗号分隔。

例如， alert()函数使一个弹出框出现在浏览器窗口内，但是我们需要给它一个字符串作为参数，以告诉该函数显示什么消息。

您也可以定义自己的功能。在下一个示例中，我们创建一个简单的函数，该函数将两个数字作为参数并将它们相乘： 

function multiply(num1,num2) {
  let result = num1 * num2;
  return result;
}

尝试在控制台中运行它；然后测试几个参数。例如： 

multiply(4, 7);
multiply(20, 20);
multiply(0.5, 3);

注意： return语句告诉浏览器将result变量从函数中返回，以便可以使用。这是必需的，因为在函数内部定义的变量仅在这些函数内部可用。这称为变量{{Glossary("Scope", "scoping")}}。 （了解有关变量作用域的更多信息。）

大事记

网站上的真正互动需要事件处理程序。这些是在浏览器中侦听活动并作为响应运行代码的代码结构。最明显的示例是处理click事件，当您用鼠标单击某些内容时，浏览器会触发该事件。为了说明这一点，请在控制台中输入以下内容，然后单击当前网页： 

document.querySelector('html').onclick = function() {
    alert('Ouch! Stop poking me!');
}

将事件处理程序附加到元素的方法有很多。在这里我们选择{{htmlelement("html")}}元素，将其onclick处理程序属性设置为等于匿名（即，无名）函数，该函数包含我们希望click事件运行的代码。

注意

document.querySelector('html').onclick = function() {};

相当于

let myHTML = document.querySelector('html');
myHTML.onclick = function() {};

它只是更短。

添加图像转换器

在本部分中，您将学习如何使用JavaScript和DOM API功能来交替显示两个图像之一。当用户单击显示的图像时，将发生此更改。

  1. 选择您要在示例网站上展示的图片。理想情况下，图像的大小应与之前添加的图像相同，或尽可能接近。
  2. 将此图像保存在您的images文件夹中。
  3. 重命名图像firefox2.png 。
  4. 将下面的JavaScript添加到您的main.js文件中。 （还可以从前面的练习中删除Hello world！ JavaScript。）
     let myImage = document.querySelector('img');

myImage.onclick = function() {
    let mySrc = myImage.getAttribute('src');
    if(mySrc === 'images/firefox-icon.png') {
      myImage.setAttribute('src','images/firefox2.png');
    } else {
      myImage.setAttribute('src','images/firefox-icon.png');
    }
}
  5. 保存所有文件，然后在浏览器中加载index.html 。现在，当您单击图像时，它应该更改为另一幅图像。

就是这样您存储了对您的参考{{htmlelement("img")}}myImage变量中的元素。接下来，使此变量的onclick事件处理程序属性等于没有名称的函数（“匿名”函数）。因此，每次单击此元素：

  1. 该代码检索图像的src属性的值。
  2. 该代码使用条件检查src值是否等于原始图像的路径：
    1. 如果是的话，代码会将src值更改为第二张图片的路径，从而迫使另一张图片加载到{{htmlelement("img")}}元件。
    2. 如果不是（表示必须已经更改），则src值会交换回原始图像路径，并恢复为原始状态。

添加个性化欢迎消息

接下来，让我们在用户首次访问该网站时将页面标题更改为个性化的欢迎消息。此欢迎消息将继续存在。如果用户离开站点并稍后返回，我们将使用Web Storage API保存消息。我们还将包括一个用于更改用户的选项，因此也提供了欢迎消息。

  1. 在index.html ， index.html下行添加到{{htmlelement("script")}}元件： 
    <button>Change user</button>
  2. 在main.js ，将以下代码准确地放在文件底部。这将引用新按钮和标题，并存储每个内部变量：
     let myButton = document.querySelector('button');
let myHeading = document.querySelector('h1');
  3. 添加以下功能来设置个性化问候语。目前尚无任何作用，但很快就会改变。 
    function setUserName() {
  let myName = prompt('Please enter your name.');
  localStorage.setItem('name', myName);
  myHeading.textContent = 'Mozilla is cool, ' + myName;
}
    所述setUserName()函数包含一个prompt()函数，它显示一个对话框，类似于alert()这个alert() prompt()函数的作用不只是alert() ，它还要求用户输入数据，然后在用户单击OK之后将其存储在变量中。在这种情况下，我们要求用户输入名称。接下来，代码调用API localStorage ，这使我们可以将数据存储在浏览器中并在以后检索。我们使用localStorage的setItem()函数创建和存储名为'name'的数据项，并将其值设置为myName变量，该变量包含用户输入的名称。最后，我们将标题的textContent设置为字符串，再加上用户新存储的名称。
  4. 在下面添加if ... else块。我们可以调用此初始化代码，因为它在首次加载时会构建应用程序。 
    if(!localStorage.getItem('name')) {
  setUserName();
} else {
  let storedName = localStorage.getItem('name');
  myHeading.textContent = 'Mozilla is cool, ' + storedName;
}
    该块的第一行使用否定运算符（逻辑NOT，由!表示）检查name数据是否存在。如果不是，则运行setUserName()函数来创建它。如果存在（即用户在上一次访问期间设置了用户名），我们将使用getItem()检索存储的名称，并将标题的textContent设置为字符串，再加上用户名，就像在setUserName()所做的setUserName() 。
  5. 将此onclick事件处理程序（如下）放在按钮上。单击后， setUserName()运行setUserName() 。这允许用户通过按下按钮输入其他名称。 
    myButton.onclick = function() {
  setUserName();
}

用户名是否为null？

当您运行该示例并获得提示您输入用户名的对话框时，请尝试按“取消”按钮。您应该以一个标题结尾，该标题显示Mozilla很酷，null 。发生这种情况的原因是-当您取消提示时-该值设置为null 。 Null是JavaScript中的一个特殊值，它表示缺少值。

另外，尝试单击“确定”而不输入名称。由于相当明显的原因，您应该以一个标题结尾，该标题的内容应为Mozilla很棒。

为避免这些问题，您可以检查用户是否输入了空白名称。将您的setUserName()函数更新为： 

function setUserName() {
  let myName = prompt('Please enter your name.');
  if(!myName) {
    setUserName();
  } else {
    localStorage.setItem('name', myName);
    myHeading.innerHTML = 'Mozilla is cool, ' + myName;
  }
}

用人类语言来说，这意味着：如果myName没有值，请setUserName()开始再次运行setUserName() 。如果它确实有一个值（如果上面的语句不正确），则将该值存储在localStorage并将其设置为标题的文本。
